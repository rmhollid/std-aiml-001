# 2. PURPOSE_SCOPE
This document defines a standards-grade study dictionary, glossary, concept map, and proof reference for artificial intelligence and machine learning.

The publisher / standards authority label for this document is STSF_GOVERNANCE_AUTHORITY.

The purpose is to help a student understand AI and ML by organizing terms by functional role rather than alphabet alone.

The document is intended to be read as a guided study manual, not as a wall of disconnected definitions. A beginner should read the plain definition, job, example, and common confusion before reading formulas or proofs.

This document covers:
- data and representation
- vectors, tensors, matrices, and spaces
- probability and statistics
- optimization and learning
- supervised learning
- unsupervised learning
- neural networks
- transformers and attention
- language models
- reinforcement learning
- evaluation and validation
- deployment and inference
- core proof summaries used in AI and ML study

This document does not claim to be a complete encyclopedia. It is a structured foundation intended for expansion.

# 3. DEFINITIONS
AI
Artificial intelligence. A field concerned with systems that perform tasks normally associated with reasoning, perception, planning, language, decision-making, or adaptive behavior.

ML
Machine learning. A subfield of AI in which systems improve task performance by fitting patterns from data rather than being fully hand-programmed.

Term
A named word or phrase used in AI/ML study.

Concept
A reusable idea that explains how multiple terms, models, or formulas connect.

Formula
A symbolic statement defining a relationship, update, objective, or measurement.

Proof Summary
A compressed derivation showing why a statement follows from stated assumptions.

Functional Role
The job a term performs in the AI/ML system chain.

Model
A parameterized function or rule system that maps inputs to outputs.

Parameter
A learned internal value of a model, such as a weight or bias.

Hyperparameter
A configuration value chosen outside the training update process, such as learning rate, batch size, layer count, or regularization strength.

Dataset
A collection of examples used for training, validation, testing, or inference.

Feature
A measurable input property used by a model.

Label
The target value associated with a supervised training example.

Loss Function
A function that measures disagreement between model output and desired output.

Objective Function
The function a learning algorithm attempts to minimize or maximize.

Gradient
A vector of partial derivatives indicating the direction of steepest increase of a scalar function.

Training
The process of adjusting parameters to reduce loss or increase reward.

Inference
The process of using a trained model to produce outputs from new inputs.

Generalization
A model's ability to perform well on data not seen during training.

Overfitting
A failure mode where the model learns training-specific noise or accidental patterns and performs poorly on unseen data.

Underfitting
A failure mode where the model is too weak, poorly trained, or poorly specified to capture the useful structure in the data.

Embedding
A learned or constructed vector representation of an object such as a token, image patch, document, user, item, state, or action.

Latent Space
A representation space where hidden structure is encoded in vectors not directly equal to raw input values.

Attention
A mechanism that computes context-sensitive weighted combinations of representations.

Transformer
A neural architecture based primarily on attention, feedforward layers, normalization, and residual connections.

Token
A discrete unit of text or data used by a model.

Tokenizer
A system that converts raw text or other input into tokens and token identifiers.


Plain-Language Bridge
A short restatement that explains a technical term using ordinary language before or beside the formal wording.

Reader Path
A guided order for reading the document so a beginner is not forced to understand advanced material before foundational material.

Study Chunk
A small unit of learning that contains one idea, one plain meaning, one role, one example, and one confusion warning.

Accessibility Constraint
A rule that technical correctness must be preserved while reducing unnecessary reading difficulty.

# 4. SYSTEM_OVERVIEW
The glossary is organized as a function chain:

raw data -> representations -> probability -> model family -> training -> optimization -> evaluation -> inference -> deployment

The core learning idea is:

1. A real system produces observations.
2. Observations are encoded as data.
3. Data is represented in a mathematical space.
4. A model maps inputs to outputs.
5. A loss or reward measures quality.
6. An optimizer changes parameters.
7. Evaluation estimates behavior on unseen cases.
8. Inference applies the fitted model.

The document separates four knowledge layers:

Layer A: vocabulary.
Layer B: concepts.
Layer C: formulas.
Layer D: proof summaries and derivations.


## 4.1 HUMAN READING MODEL
This document is a technical standard, but it is also a study manual. A reader should not have to already know AI or ML to begin using it.

The human reading order is:

1. Read the plain meaning first.
2. Read the functional role second.
3. Read the example third.
4. Read the formula only after the idea is visible.
5. Read the proof only after the formula is understood.

The simplest version of the learning chain is:

- Data is what the system learns from.
- A feature is one useful piece of input information.
- A model is the rule being trained.
- A parameter is a learned setting inside the model.
- A loss tells the model how wrong it was.
- A gradient tells which way to change.
- Training changes the parameters.
- Evaluation checks whether learning worked on new examples.
- Inference uses the trained model.

A beginner may skip proofs on first reading. Proofs are present to show why formulas are valid, not to block basic understanding.

# 8. ALGORITHMIC_FORMALISM
## 8.1 BASIC LEARNING LOOP
Given data D = {(x_i, y_i)} for i = 1..n:

1. Choose a model f_theta.
2. Choose a loss L(y, f_theta(x)).
3. Define empirical risk:
   R_hat(theta) = (1/n) sum_i L(y_i, f_theta(x_i)).
4. Initialize theta.
5. Repeat:
   theta <- theta - eta * gradient_theta R_hat(theta).
6. Validate on held-out data.
7. Use the final theta for inference.

## 8.2 CORE FORMULA LIST
Linear model:
  y_hat = w^T x + b

Mean squared error:
  MSE = (1/n) sum_i (y_i - y_hat_i)^2

Binary logistic prediction:
  p(y=1|x) = 1 / (1 + exp(-(w^T x + b)))

Softmax:
  softmax(z)_i = exp(z_i) / sum_j exp(z_j)

Cross entropy:
  H(p, q) = - sum_i p_i log(q_i)

KL divergence:
  KL(p||q) = sum_i p_i log(p_i/q_i)

Gradient descent:
  theta_{t+1} = theta_t - eta * gradient_theta J(theta_t)

Dot-product attention:
  Attention(Q,K,V) = softmax(QK^T / sqrt(d_k)) V

Bayes theorem:
  P(A|B) = P(B|A)P(A) / P(B)

Expected value:
  E[X] = sum_x x P(X=x)

Variance:
  Var(X) = E[(X - E[X])^2]

Language-model factorization:
  P(x_1,...,x_n) = product_i P(x_i | x_<i)

Bellman value recursion:
  V^pi(s) = E_pi[r_t + gamma V^pi(s_{t+1}) | s_t=s]

## 8.3 PROOF APPENDIX
PROOF_ID: AIML-PROOF-0001
TITLE: MSE gradient for linear regression
STATEMENT: For y_hat=Xw+b and MSE=(1/n)||Xw+b-y||_2^2, gradient_w MSE=(2/n)X^T(Xw+b-y).
ASSUMPTIONS: X is n by d; w is d by 1; b is broadcast or included as an intercept column; y is n by 1.
DERIVATION_STEPS:
  1. Let e=Xw+b-y.
  2. MSE=(1/n)e^T e.
  3. The differential is d(MSE)=(2/n)e^T de.
  4. Since de=X dw, d(MSE)=(2/n)e^T X dw.
  5. Rewrite as d(MSE)=((2/n)X^T e)^T dw.
  6. Therefore gradient_w MSE=(2/n)X^T e.
CONCLUSION: For y_hat=Xw+b and MSE=(1/n)||Xw+b-y||_2^2, gradient_w MSE=(2/n)X^T(Xw+b-y).
AI_ML_RELEVANCE: This is the canonical first derivation for understanding gradient-based supervised learning.

PROOF_ID: AIML-PROOF-0002
TITLE: Softmax outputs sum to one
STATEMENT: For softmax(z)_i=exp(z_i)/sum_j exp(z_j), outputs are positive and sum to one.
ASSUMPTIONS: z is finite; denominator is nonzero because exp(z_j)>0.
DERIVATION_STEPS:
  1. Let S=sum_j exp(z_j).
  2. Each exp(z_i)>0, so each output is positive.
  3. sum_i softmax(z)_i=sum_i exp(z_i)/S.
  4. This equals (sum_i exp(z_i))/S=S/S=1.
CONCLUSION: For softmax(z)_i=exp(z_i)/sum_j exp(z_j), outputs are positive and sum to one.
AI_ML_RELEVANCE: Explains why logits can be converted to class or token probabilities.

PROOF_ID: AIML-PROOF-0003
TITLE: Cross entropy minimized by matching the target distribution
STATEMENT: For fixed p, H(p,q) is minimized when q=p.
ASSUMPTIONS: p and q are valid distributions; q_i>0 when p_i>0.
DERIVATION_STEPS:
  1. Use H(p,q)=H(p)+KL(p||q).
  2. H(p) is constant with respect to q.
  3. KL(p||q)>=0 by Gibbs' inequality.
  4. KL(p||q)=0 only when p=q.
  5. Therefore H(p,q) is minimized at q=p.
CONCLUSION: For fixed p, H(p,q) is minimized when q=p.
AI_ML_RELEVANCE: Shows why cross-entropy training pushes predicted distributions toward target distributions.

PROOF_ID: AIML-PROOF-0004
TITLE: KL divergence is nonnegative
STATEMENT: KL(p||q)>=0 for valid distributions p and q.
ASSUMPTIONS: p_i>=0, q_i>0 where p_i>0, and both distributions sum to one.
DERIVATION_STEPS:
  1. Use the convexity of -log or Jensen's inequality.
  2. KL(p||q)=sum_i p_i log(p_i/q_i).
  3. This is the expected log ratio under p.
  4. Gibbs' inequality gives the lower bound 0.
  5. Equality holds only when p=q on the support of p.
CONCLUSION: KL(p||q)>=0 for valid distributions p and q.
AI_ML_RELEVANCE: Supports cross entropy decomposition and many probabilistic learning objectives.

PROOF_ID: AIML-PROOF-0005
TITLE: Negative gradient gives steepest local descent
STATEMENT: For small step epsilon under Euclidean norm, moving along -gradient J gives the largest first-order decrease.
ASSUMPTIONS: J is differentiable at theta; step vector u has ||u||=1.
DERIVATION_STEPS:
  1. First-order Taylor expansion: J(theta+epsilon u) approx J(theta)+epsilon gradient J dot u.
  2. To minimize the linear term, minimize gradient J dot u over unit u.
  3. By Cauchy-Schwarz, gradient J dot u >= -||gradient J||.
  4. Equality occurs when u=-gradient J/||gradient J||.
  5. Therefore the negative gradient is steepest local descent.
CONCLUSION: For small step epsilon under Euclidean norm, moving along -gradient J gives the largest first-order decrease.
AI_ML_RELEVANCE: Explains the geometric reason behind gradient descent.

PROOF_ID: AIML-PROOF-0006
TITLE: Linear layers without nonlinearities collapse to one linear layer
STATEMENT: A stack of affine layers without nonlinear activations is equivalent to one affine layer.
ASSUMPTIONS: Each layer has h_{l+1}=W_l h_l+b_l.
DERIVATION_STEPS:
  1. Compose two affine maps: W2(W1x+b1)+b2.
  2. This equals (W2W1)x+(W2b1+b2).
  3. The result is still affine.
  4. Repeating this argument collapses any finite stack to one affine map.
CONCLUSION: A stack of affine layers without nonlinear activations is equivalent to one affine layer.
AI_ML_RELEVANCE: Explains why activation functions are required for nonlinear neural networks.

PROOF_ID: AIML-PROOF-0007
TITLE: Attention weights are normalized row-wise
STATEMENT: In dot-product attention, each query row's softmax scores sum to one.
ASSUMPTIONS: QK^T/sqrt(d_k) is finite; softmax is applied across keys per query.
DERIVATION_STEPS:
  1. Each row contains scores comparing one query to all keys.
  2. Softmax maps that row to positive weights summing to one.
  3. The output row equals a weighted sum of value vectors.
  4. Therefore each query receives a normalized context mixture.
CONCLUSION: In dot-product attention, each query row's softmax scores sum to one.
AI_ML_RELEVANCE: Explains the weighted-average structure of transformer attention.

PROOF_ID: AIML-PROOF-0008
TITLE: Language-model sequence probability factorization
STATEMENT: Any joint sequence probability can be factored into conditional next-token probabilities.
ASSUMPTIONS: The sequence has finite length and all conditioning events have nonzero probability where required.
DERIVATION_STEPS:
  1. Apply the probability chain rule.
  2. P(x_1,...,x_n)=P(x_1)P(x_2|x_1)...P(x_n|x_1,...,x_{n-1}).
  3. Write x_<i for the prefix before token i.
  4. The product becomes product_i P(x_i|x_<i).
CONCLUSION: Any joint sequence probability can be factored into conditional next-token probabilities.
AI_ML_RELEVANCE: Justifies next-token prediction as a way to model full sequence probability.

PROOF_ID: AIML-PROOF-0009
TITLE: Variance identity
STATEMENT: Var(X)=E[X^2]-(E[X])^2.
ASSUMPTIONS: X has finite second moment.
DERIVATION_STEPS:
  1. Let mu=E[X].
  2. Var(X)=E[(X-mu)^2].
  3. Expand the square: E[X^2-2mu X+mu^2].
  4. Use linearity: E[X^2]-2mu E[X]+mu^2.
  5. Since E[X]=mu, this becomes E[X^2]-mu^2.
CONCLUSION: Var(X)=E[X^2]-(E[X])^2.
AI_ML_RELEVANCE: Used in statistics, uncertainty estimation, and bias-variance analysis.

PROOF_ID: AIML-PROOF-0010
TITLE: Bias-variance decomposition for squared error
STATEMENT: Expected squared prediction error decomposes into noise, bias squared, and variance.
ASSUMPTIONS: Squared loss; target y=f(x)+epsilon; E[epsilon]=0; epsilon independent of model sampling.
DERIVATION_STEPS:
  1. Consider E[(y-f_hat(x))^2].
  2. Substitute y=f(x)+epsilon.
  3. Separate irreducible noise from model error.
  4. Add and subtract E[f_hat(x)].
  5. Expand the square and take expectation.
  6. Cross terms vanish under assumptions.
  7. Result: noise + bias^2 + variance.
CONCLUSION: Expected squared prediction error decomposes into noise, bias squared, and variance.
AI_ML_RELEVANCE: Explains why both underfitting and overfitting can increase error.

PROOF_ID: AIML-PROOF-0011
TITLE: Bellman value recursion
STATEMENT: The value of a policy equals expected immediate reward plus discounted next-state value.
ASSUMPTIONS: Markov decision process; fixed policy pi; discounted return G_t=sum_k gamma^k r_{t+k}.
DERIVATION_STEPS:
  1. Write G_t=r_t+gamma r_{t+1}+gamma^2 r_{t+2}+...
  2. Group the tail: G_t=r_t+gamma G_{t+1}.
  3. Take expectation conditioned on s_t=s under policy pi.
  4. Obtain V^pi(s)=E_pi[r_t+gamma V^pi(s_{t+1})|s_t=s].
CONCLUSION: The value of a policy equals expected immediate reward plus discounted next-state value.
AI_ML_RELEVANCE: Defines the recursive core of reinforcement learning.

PROOF_ID: AIML-PROOF-0012
TITLE: L2 regularization shrinks weights in the gradient update
STATEMENT: Adding lambda||w||^2 to loss adds a gradient term proportional to w.
ASSUMPTIONS: Objective J(w)=L(w)+lambda||w||_2^2.
DERIVATION_STEPS:
  1. Differentiate the penalty: gradient_w lambda||w||^2 = 2lambda w.
  2. Total gradient is gradient L(w)+2lambda w.
  3. Gradient descent update becomes w <- w - eta gradient L(w) - 2eta lambda w.
  4. The last term multiplies current weights by a factor below one when eta lambda is positive.
CONCLUSION: Adding lambda||w||^2 to loss adds a gradient term proportional to w.
AI_ML_RELEVANCE: Explains weight decay as a simple regularization mechanism.

CATEGORY: CAT-MATH

PLAIN_LANGUAGE:
A probability distribution spreads total probability across possible outcomes. In a language model, it can describe how likely each next token is.

CATEGORY: CAT-TRAIN

PLAIN_LANGUAGE:
Cross entropy is a loss for predicted probabilities. It gets larger when the model gives low probability to the correct answer.

## 14.5 WORKED EXAMPLES
EXAMPLE_ID: AIML-EXAMPLE-0001
TITLE: Linear regression one-step gradient update
PURPOSE: Show how a model parameter changes from data, loss, and gradient.
GIVEN: One input x=2, target y=5, model y_hat=w*x+b, w=1, b=0, learning rate eta=0.1.
STEPS:
  1. Predict y_hat=1*2+0=2.
  2. Error e=y_hat-y=2-5=-3.
  3. Squared loss L=e^2=9.
  4. Gradient with respect to w is dL/dw=2e*x=2*(-3)*2=-12.
  5. Gradient with respect to b is dL/db=2e=-6.
  6. Update w_new=w-eta*dL/dw=1-0.1*(-12)=2.2.
  7. Update b_new=b-eta*dL/db=0-0.1*(-6)=0.6.
RESULT: The model moves from y_hat=2 toward y=5.
RELATED_TERMS: AIML-TERM-0030 Linear Regression, AIML-TERM-0042 Mean Squared Error, AIML-TERM-0043 Gradient Descent

EXAMPLE_ID: AIML-EXAMPLE-0002
TITLE: Logistic regression probability calculation
PURPOSE: Show how a linear score becomes a probability.
GIVEN: z=w^T x+b=1.5.
STEPS:
  1. Use sigmoid p=1/(1+exp(-z)).
  2. Substitute z=1.5.
  3. p=1/(1+exp(-1.5)).
  4. exp(-1.5) is approximately 0.223.
  5. p is approximately 1/(1.223)=0.818.
RESULT: The model predicts about 81.8 percent probability for class 1.
RELATED_TERMS: AIML-TERM-0031 Logistic Regression, AIML-TERM-0022 Cross Entropy, AIML-TERM-0074 Calibration

EXAMPLE_ID: AIML-EXAMPLE-0003
TITLE: Softmax three-class normalization
PURPOSE: Show how logits become a probability distribution.
GIVEN: logits z=[2,1,0].
STEPS:
  1. Compute exponentials exp(z)=[exp(2),exp(1),exp(0)].
  2. Approximate exp(z)=[7.389,2.718,1].
  3. Sum S=7.389+2.718+1=11.107.
  4. Divide each exponential by S.
  5. softmax(z) is approximately [0.665,0.245,0.090].
RESULT: The three class probabilities are positive and sum to approximately 1.
RELATED_TERMS: AIML-TERM-0056 Attention, AIML-TERM-0022 Cross Entropy, AIML-TERM-0075 Temperature

EXAMPLE_ID: AIML-EXAMPLE-0004
TITLE: Attention weighted-sum calculation
PURPOSE: Show that attention output is a weighted mixture of value vectors.
GIVEN: One query produces weights [0.70,0.20,0.10] over value scalars [10,20,50].
STEPS:
  1. Multiply each value by its attention weight.
  2. 0.70*10=7.
  3. 0.20*20=4.
  4. 0.10*50=5.
  5. Sum weighted values: 7+4+5=16.
RESULT: The attention output for that query is 16.
RELATED_TERMS: AIML-TERM-0056 Attention, AIML-TERM-0057 Query-Key-Value, AIML-TERM-0058 Transformer

EXAMPLE_ID: AIML-EXAMPLE-0005
TITLE: Train-validation-test split use
PURPOSE: Separate fitting, tuning, and final estimation.
GIVEN: 1000 labeled samples.
STEPS:
  1. Assign 700 samples to training.
  2. Assign 150 samples to validation.
  3. Assign 150 samples to testing.
  4. Fit parameters using training samples only.
  5. Choose hyperparameters using validation results.
  6. Report final performance once using the test set.
RESULT: The test estimate is less contaminated by development choices.
RELATED_TERMS: AIML-TERM-0006 Training Set, AIML-TERM-0007 Validation Set, AIML-TERM-0008 Test Set

TITLE: Overfitting recognition pattern
PURPOSE: Detect the common gap between training and validation behavior.
GIVEN: Training accuracy=99 percent; validation accuracy=72 percent.
STEPS:
  1. Compare performance on fitted data and held-out data.
  2. Large training performance with much lower validation performance indicates poor generalization.
  3. Candidate repairs include more data, regularization, simpler model, early stopping, or better split design.

PROOF_ID: AIML-PROOF-0013
TITLE: Gibbs Inequality and KL Nonnegativity
STATEMENT: For probability distributions p and q over the same finite support with q_i > 0 wherever p_i > 0, KL(p||q) >= 0, with equality when p=q on the support.
ASSUMPTIONS: p_i >= 0, q_i > 0 on p support, sum_i p_i = sum_i q_i = 1, and log is natural logarithm.
DERIVATION:
1. Use the scalar inequality log x <= x - 1 for x > 0, with equality only when x=1.
2. Substitute x = q_i / p_i for every term with p_i > 0.
3. log(q_i/p_i) <= q_i/p_i - 1.
4. Multiply by p_i: p_i log(q_i/p_i) <= q_i - p_i.
5. Sum over i: sum_i p_i log(q_i/p_i) <= sum_i q_i - sum_i p_i = 1 - 1 = 0.
6. Multiply by -1: sum_i p_i log(p_i/q_i) >= 0.
7. Equality requires q_i/p_i = 1 for all p_i > 0, so p_i=q_i on the support.
AI_ML_RELEVANCE: KL divergence measures mismatch between a target distribution and predicted distribution; nonnegativity makes it an optimization penalty.

PROOF_ID: AIML-PROOF-0014
TITLE: Cross Entropy Decomposition
STATEMENT: H(p,q) = H(p) + KL(p||q), so cross entropy is minimized when q=p under a free distribution choice.
ASSUMPTIONS: p and q are finite distributions with q_i > 0 wherever p_i > 0.
DERIVATION:
1. Start with cross entropy H(p,q) = -sum_i p_i log q_i.
2. Add and subtract log p_i inside the sum: -sum_i p_i log q_i = -sum_i p_i log p_i + sum_i p_i log(p_i/q_i).
3. The first term is H(p).
4. The second term is KL(p||q).
5. Therefore H(p,q)=H(p)+KL(p||q).
6. H(p) does not depend on q.
7. By Gibbs inequality, KL(p||q)>=0 and equals zero when q=p.
8. Therefore H(p,q) is minimized by q=p.
AI_ML_RELEVANCE: Explains why minimizing cross entropy trains a classifier to match the target distribution.

PROOF_ID: AIML-PROOF-0015
TITLE: Bias-Variance Decomposition
STATEMENT: For squared-error prediction at fixed x, E[(Y - h(X))^2] decomposes into irreducible noise, squared bias, and variance of the estimator.
ASSUMPTIONS: Let Y=f(x)+epsilon, E[epsilon]=0, Var(epsilon)=sigma^2, and h is a trained predictor depending on sampled training data.
DERIVATION:
1. At fixed x, define mu_h = E[h(x)], where expectation is over training sets.
2. Error is Y-h(x)=f(x)+epsilon-h(x).
3. Add and subtract mu_h: Y-h(x) = epsilon + f(x)-mu_h + mu_h-h(x).
4. Square and take expectation.
5. E[(Y-h)^2] = E[epsilon^2] + (f-mu_h)^2 + E[(h-mu_h)^2] plus cross terms.
6. Cross term E[epsilon(f-mu_h)] = (f-mu_h)E[epsilon]=0.
7. Cross term E[epsilon(mu_h-h)] = E[epsilon]E[mu_h-h]=0 when noise is independent of training-set variation.
8. Cross term E[(f-mu_h)(mu_h-h)] = (f-mu_h)(mu_h-E[h])=0.
9. Therefore E[(Y-h)^2] = sigma^2 + Bias(h)^2 + Var(h).
AI_ML_RELEVANCE: Explains underfitting, overfitting, and why model complexity creates a tradeoff.

PROOF_ID: AIML-PROOF-0016
TITLE: Gradient Direction Is Steepest Ascent
STATEMENT: For differentiable scalar f, the unit direction that maximizes the directional derivative is the gradient direction.
ASSUMPTIONS: f differentiable at x, gradient not zero, and ||u||_2=1.
DERIVATION:
1. Directional derivative in direction u is D_u f(x)=grad f(x) dot u.
2. By Cauchy-Schwarz, grad f dot u <= ||grad f|| ||u||.
3. Since ||u||=1, D_u f(x) <= ||grad f||.
4. Equality occurs when u is parallel to grad f.
5. Therefore the steepest increasing unit direction is u=grad f/||grad f||.
6. The steepest decreasing unit direction is -grad f/||grad f||.
AI_ML_RELEVANCE: Justifies using the negative gradient direction to reduce loss.

PROOF_ID: AIML-PROOF-0017
TITLE: Gradient Descent Descent Property
STATEMENT: If f has L-Lipschitz gradient and 0 < eta <= 1/L, then a gradient descent step does not increase f and decreases it unless the gradient is zero.
ASSUMPTIONS: f is differentiable and L-smooth.
DERIVATION:
1. L-smoothness gives f(y) <= f(x)+grad f(x)^T(y-x)+(L/2)||y-x||^2.
2. Let y=x-eta grad f(x).
3. Then y-x = -eta grad f(x).
4. Substitute: f(y) <= f(x) - eta||grad f(x)||^2 + (L/2)eta^2||grad f(x)||^2.
5. Combine terms: f(y) <= f(x) - eta(1 - L eta/2)||grad f(x)||^2.
6. If eta <= 1/L, then 1 - L eta/2 >= 1/2.
7. Therefore f(y) <= f(x) - (eta/2)||grad f(x)||^2.
8. If grad f(x) is nonzero, the right side is strictly less than f(x).
AI_ML_RELEVANCE: Gives the mathematical reason learning rates must be small enough.

PROOF_ID: AIML-PROOF-0018

PROOF_ID: AIML-PROOF-0020
TITLE: Basis Coordinate Uniqueness

PROOF_ID: AIML-PROOF-0021
TITLE: Variance Moment Identity
STATEMENT: Var(X)=E[X^2]-(E[X])^2.
ASSUMPTIONS: X has finite second moment and mu=E[X].
DERIVATION:
1. Var(X)=E[(X-mu)^2].
2. Expand the square: (X-mu)^2 = X^2 - 2mu X + mu^2.
3. Apply linearity of expectation: E[X^2 - 2mu X + mu^2] = E[X^2] - 2mu E[X] + mu^2.
4. Since mu=E[X], substitute: E[X^2] - 2mu^2 + mu^2.
5. Therefore Var(X)=E[X^2]-mu^2=E[X^2]-(E[X])^2.
AI_ML_RELEVANCE: Used in normalization, uncertainty, and bias-variance analysis.

PROOF_ID: AIML-PROOF-0022
TITLE: Convex Local Minimum Is Global
STATEMENT: Any local minimum of a convex function over a convex domain is a global minimum.
ASSUMPTIONS: f is convex on convex domain C; x* is a local minimum.
DERIVATION:
1. Assume for contradiction that x* is not global.
2. Then there exists y in C with f(y)<f(x*).
3. For lambda in (0,1), z_lambda=(1-lambda)x*+lambda y lies in C.
4. By convexity, f(z_lambda) <= (1-lambda)f(x*) + lambda f(y).
5. Since f(y)<f(x*), the right side is less than f(x*).
6. For sufficiently small lambda, z_lambda is arbitrarily close to x*.
7. This contradicts x* being a local minimum.
8. Therefore x* is global.
AI_ML_RELEVANCE: Explains why convex objectives are easier than deep non-convex losses.

EXAMPLE_ID: AIML-EXAMPLE-0007
TITLE: Gradient Descent Numerical Update
SETUP: Let J(w)=(w-3)^2, starting w_0=0, learning rate eta=0.1.
STEPS:
1. dJ/dw = 2(w-3).
2. At w_0=0, gradient = -6.
3. Update w_1 = w_0 - eta * gradient = 0 - 0.1*(-6)=0.6.
4. New loss J(0.6)=(-2.4)^2=5.76, lower than J(0)=9.
LESSON: A negative gradient means increasing w reduces the loss.

TITLE: Matrix Multiplication as a Neural Layer

2. Add bias: z=Wx+b=[9,1]^T.
3. Apply activation if needed, such as ReLU(z)=[9,1]^T.

EXAMPLE_ID: AIML-EXAMPLE-0009
TITLE: Bayes Theorem Calculation
SETUP: P(Disease)=0.01, P(Positive|Disease)=0.99, P(Positive|No Disease)=0.05.
STEPS:
1. P(Positive)=0.99*0.01 + 0.05*0.99 = 0.0594.
2. P(Disease|Positive)=0.99*0.01 / 0.0594.
3. Result = 0.1667 approximately.
LESSON: A strong test can still yield modest posterior probability when the base rate is low.

EXAMPLE_ID: AIML-EXAMPLE-0010
TITLE: Precision Recall and F1
SETUP: TP=40, FP=10, FN=20, TN=930.
STEPS:
1. Precision = TP/(TP+FP)=40/50=0.8.
2. Recall = TP/(TP+FN)=40/60=0.6667.
3. F1 = 2PR/(P+R)=2*0.8*0.6667/(1.4667)=0.7273.
LESSON: Precision measures correctness among positive predictions; recall measures captured positives.

EXAMPLE_ID: AIML-EXAMPLE-0011
TITLE: Embedding Cosine Similarity
SETUP: a=[1,1], b=[2,0].
STEPS:
1. Dot product a dot b = 2.
2. Norms are ||a||=sqrt(2), ||b||=2.
3. Cosine similarity = 2/(2sqrt(2)) = 1/sqrt(2) = 0.7071.
LESSON: Cosine similarity measures angle-like alignment rather than raw magnitude alone.

EXAMPLE_ID: AIML-EXAMPLE-0012
TITLE: One-Step Backpropagation Mini-Example
SETUP: y_hat=wx, loss L=(y_hat-y)^2, with x=2, y=5, w=1.
STEPS:
1. Forward: y_hat=1*2=2.
2. Loss: L=(2-5)^2=9.
3. dL/dy_hat = 2(y_hat-y)=2(2-5)=-6.
4. dy_hat/dw = x = 2.
5. Chain rule: dL/dw = dL/dy_hat * dy_hat/dw = -12.
6. With eta=0.1, w_new = 1 - 0.1*(-12)=2.2.
LESSON: Backpropagation is chain rule through the computation graph.

```python
from math import sqrt

def dot(a, b):
    return sum(x * y for x, y in zip(a, b))

def norm(a):
    return sqrt(dot(a, a))

def cosine(a, b):
    return dot(a, b) / (norm(a) * norm(b))
```

## 14.17 V0.1.5 ADDITIONAL PROOF AND DERIVATION TARGETS
PROOF_ID: AIML-PROOF-0023
TITLE: F1 score from precision and recall
STATEMENT: F1 = 2PR/(P+R), where P is precision and R is recall.
ASSUMPTIONS: P+R>0; precision and recall are nonnegative.
DERIVATION_STEPS:
  1. F1 is defined as the harmonic mean of precision and recall.
  2. The harmonic mean of two positive values a and b is 2/(1/a+1/b).
  3. Substitute a=P and b=R.
  4. F1=2/(1/P+1/R)=2/(R/(PR)+P/(PR)).
  5. Therefore F1=2PR/(P+R).
CONCLUSION: F1 combines precision and recall using their harmonic mean.
AI_ML_RELEVANCE: Explains why F1 is low when either precision or recall is low.

PROOF_ID: AIML-PROOF-0024
TITLE: Softmax cross-entropy gradient with logits
STATEMENT: For one-hot target y and p=softmax(z), gradient_z CE = p-y.
ASSUMPTIONS: y is one-hot or a valid target distribution; p_i=exp(z_i)/sum_j exp(z_j); CE=-sum_i y_i log p_i.
DERIVATION_STEPS:
  1. log p_i = z_i - log(sum_j exp(z_j)).
  2. CE = -sum_i y_i z_i + sum_i y_i log(sum_j exp(z_j)).
  3. If sum_i y_i=1, CE = -sum_i y_i z_i + log(sum_j exp(z_j)).
  4. Differentiate with respect to z_k: derivative is -y_k + exp(z_k)/sum_j exp(z_j).
  5. The second term is p_k.
CONCLUSION: gradient_z CE = p-y.
AI_ML_RELEVANCE: This is the central derivative behind multiclass classification and language-model next-token training.

PROOF_ID: AIML-PROOF-0025
TITLE: K-means update step minimizes within-cluster squared distance
STATEMENT: Given fixed cluster assignments, the centroid minimizing sum of squared distances is the arithmetic mean of assigned points.
ASSUMPTIONS: Euclidean squared distance; a nonempty cluster C.
DERIVATION_STEPS:
  1. For one cluster, minimize J(mu)=sum_{x in C} ||x-mu||^2.
  2. Differentiate: gradient_mu J = 2 sum_{x in C}(mu-x).
  3. Set gradient to zero: sum_{x in C}(mu-x)=0.
  4. |C| mu = sum_{x in C} x.
  5. Therefore mu=(1/|C|)sum_{x in C}x.
CONCLUSION: The mean is the optimal centroid for fixed assignments.
AI_ML_RELEVANCE: Justifies the centroid update used by k-means.

PROOF_ID: AIML-PROOF-0026
TITLE: Attention scaling controls dot-product variance
STATEMENT: Dividing QK^T by sqrt(d_k) keeps score variance from growing linearly with key dimension under simple independent assumptions.
ASSUMPTIONS: Query/key components are independent with mean 0 and variance 1.
DERIVATION_STEPS:
  1. A dot product score is s=sum_{i=1}^{d_k} q_i k_i.
  2. Each product q_i k_i has mean 0 and variance 1 under the assumptions.
  3. The variance of the sum is d_k.
  4. Dividing by sqrt(d_k) gives Var(s/sqrt(d_k))=d_k/d_k=1.
CONCLUSION: Scaling stabilizes attention logits as dimension changes.
AI_ML_RELEVANCE: Explains the sqrt(d_k) term in transformer attention.

```python
# Linear regression one-step gradient update for y_hat = w*x + b
x = [1.0, 2.0, 3.0]
y = [2.0, 4.0, 6.0]
w, b, eta = 0.0, 0.0, 0.1
n = len(x)
```

```python
# Confusion matrix and metrics
truth = [1,1,1,0,0,0]
pred  = [1,0,1,0,1,0]
TP = sum(t==1 and p==1 for t,p in zip(truth,pred))
FP = sum(t==0 and p==1 for t,p in zip(truth,pred))
FN = sum(t==1 and p==0 for t,p in zip(truth,pred))
TN = sum(t==0 and p==0 for t,p in zip(truth,pred))
precision = TP/(TP+FP) if TP+FP else 0
recall = TP/(TP+FN) if TP+FN else 0
f1 = 2*precision*recall/(precision+recall) if precision+recall else 0
print(TP, FP, FN, TN, round(precision,3), round(recall,3), round(f1,3))
```

```python
next_best = max(Q[('s1','left')], Q[('s1','right')])
```

### 14.20.3 HUMAN-READABLE CORE LOOP
The AI/ML learning loop can be read as a simple machine process:

1. Collect examples.
2. Turn each example into input features.
3. Give the model a starting set of parameters.
4. Ask the model for a prediction.
5. Compare the prediction to the desired answer.
6. Convert the mistake into a loss value.
7. Use the gradient to decide how to change the parameters.
8. Repeat until the model improves or stops improving.
9. Test on examples not used for training.
10. Use the trained model for inference.

Formal version:
D = {(x_i, y_i)} gives examples.
f_theta(x) gives predictions.
L(y, f_theta(x)) gives the error score.
theta <- theta - eta gradient J(theta) updates the model.

Plain version:
examples -> prediction -> error -> correction -> retest -> use.

- A term should not be defined only by other advanced terms.
- Acronyms should be expanded before heavy use.
- Examples should use small numbers, familiar objects, or simple model tasks.
- The reader should be told when a topic is safe to skip on first pass.
- Dense math should be kept, but it should be surrounded by plain-language explanation.

### 14.20.5 ACCESSIBILITY VALIDATION CHECK
A human-readability pass is valid when a beginner can answer these questions for each major term group:

1. What is the thing?
2. What job does it do?
3. What goes into it?
4. What comes out of it?
5. What mistake should I avoid?

A proof section is readable enough when a beginner can identify:

1. what is being proved
2. what assumptions are required
3. what the conclusion means for AI/ML
4. whether the proof is required for first-pass study

### 14.20.8 PLAIN-LANGUAGE MINI GLOSSARY
Data: examples the system learns from.
Feature: one input clue.
Label: the answer used for supervised training.
Model: the rule or machine being trained.
Parameter: a learned setting inside the model.
Hyperparameter: a setting chosen before or outside the direct learning update.
Loss: the score that says how wrong the model was.
Gradient: the direction that tells how a small change affects loss.
Optimizer: the procedure that changes parameters.
Training: changing parameters using examples.
Validation: checking choices during development.
Testing: final checking after choices are fixed.
Inference: using the trained model.
Embedding: vector form of an object.
Attention: weighted mixing of relevant context.
Transformer: a model architecture built around attention, residual paths, normalization, and feed-forward blocks.
Calibration: whether confidence numbers match real correctness rates.
Robustness: whether behavior stays acceptable when conditions change.
Alignment: whether behavior matches intended goals and constraints.

It adds a human access layer so the same technical content can be studied in a safer order.

## 14.22 AIML READABILITY CALIBRATION EXAMPLES
AIML-TV-001 Data: A spreadsheet row qualifies as data because it records observations.
AIML-TV-002 Feature: House size can be a feature because it is one input clue.
AIML-TV-003 Label: Spam/not-spam is a label because it is the answer used during training.
AIML-TV-004 Parameter: A learned weight is a parameter because training changes it.
AIML-TV-005 Hyperparameter: Learning rate is a hyperparameter because the trainer chooses it before or outside direct parameter learning.
AIML-TV-006 Loss: A wrong prediction produces loss because the model needs an error signal.
AIML-TV-007 Gradient: A gradient gives the direction of change for reducing or increasing a quantity.
AIML-TV-008 Calibration: If 80 percent confidence predictions are correct only 50 percent of the time, the model is poorly calibrated.
AIML-TV-009 Hallucination: An invented citation is a hallucination because it is unsupported by evidence.
AIML-TV-010 Inference: Using a trained model to answer a new question is inference.
